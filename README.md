# Random Task Generator

**Автор:** Габбазов Алмаз

**Назначение:** генератор случайных задач для итоговой аттестации по курсу «Код будущего».

Установка
Вариант 1. Клонирование репозитория
bash
git clone <URL_репозитория>
cd RandomTaskGenerator
Вариант 2. Ручная загрузка
Скачайте архив с проектом с GitHub.

Распакуйте архив в нужную папку на вашем компьютере.

Перейдите в распакованную папку в терминале/командной строке.

Запуск приложения
Шаг 1. Проверка версии Python
Убедитесь, что у вас установлена подходящая версия Python:

bash
python --version
# или для Linux/macOS:
python3 --version
Ожидаемый результат: Python 3.x.x (где x — любая цифра, но версия должна быть не ниже 3.6).

Шаг 2. Запуск программы
Запустите приложение командой:

bash
python app.py
# или, если у вас несколько версий Python:
python3 app.py
Шаг 3. Первое открытие
При первом запуске программа:

автоматически создаст файлы tasks.json и history.json, если их нет;

заполнит tasks.json стандартным набором задач.

Структура проекта
RandomTaskGenerator/
├── app.py                  # Основной файл приложения (GUI на tkinter)
├── tasks.json             # Файл с задачами (категория + текст задачи)
├── history.json         # Файл с историей генераций (дата + задача)
├── README.md            # Эта документация
└── LICENSE              # Файл с текстом лицензии MIT
Описание файлов:

app.py — главный скрипт программы. Содержит:

графический интерфейс на tkinter;

логику генерации случайных задач;

функции добавления новых задач;

механизмы фильтрации по категориям;

загрузку/сохранение задач и истории.

tasks.json — хранит все доступные задачи в формате JSON. Каждая задача содержит:

"category" — категория задачи (спорт, работа, учёба, отдых);

"task" — текст задачи.

history.json — история всех сгенерированных задач. Каждая запись содержит:

дату и время генерации;

категорию задачи;

текст задачи.

README.md — текущая документация с инструкциями по запуску и использованию.

LICENSE — текст лицензии MIT, определяющий условия использования и распространения кода.

Интерфейс программы
Окно программы разделено на три функциональные зоны:

Левая часть (Добавление задач):

поле ввода для новой задачи;

список категорий (спорт, работа, учёба, отдых);

кнопка «Добавить» для сохранения новой задачи.

Центральная часть (Генерация):

кнопка «Сгенерировать» для получения случайной задачи;

область отображения результата (выбранная задача с категорией).

Правая часть (История):

список всех ранее сгенерированных задач с датой и временем.

Примеры работы
Пример 1. Генерация случайной задачи
Запустите программу (python app.py).

Не выбирая категорию, нажмите кнопку «Сгенерировать».

В центральной части окна появится случайная задача из всех доступных, например:

Учёба: Написать сочинение
Эта задача автоматически добавится в историю справа с отметкой времени.

Пример 2. Фильтрация по категории
В левой части окна выберите категорию из списка, например, «спорт».

Нажмите «Сгенерировать».

Программа покажет случайную задачу только из категории «спорт», например:

Спорт: Сделать зарядку
Задача появится в истории с указанием категории и времени генерации.

Пример 3. Добавление своей задачи
В поле ввода в левой части окна напишите новую задачу, например, «Пробежать 5 км».

Выберите категорию, например, «спорт».

Нажмите кнопку «Добавить».

Вы увидите сообщение «Задача успешно добавлена».

Новая задача теперь доступна для генерации. Проверьте это, нажав «Сгенерировать» с выбранной категорией «спорт».

Пример 4. Просмотр истории
Сгенерируйте несколько задач разными способами (с фильтрацией и без).

Посмотрите на правую часть окна — там отображается вся история в хронологическом порядке (последние сверху).

Закройте программу и запустите её снова — история автоматически загрузится из history.json.

Устранение неполадок
«Файл не найден» при запуске:

убедитесь, что вы запускаете app.py из правильной папки;

проверьте, что файлы tasks.json и history.json присутствуют (если их нет, программа создаст их автоматически).

Не добавляются новые задачи:

проверьте, что заполнены оба поля: текст задачи и выбрана категория;

убедитесь, что в тексте задачи нет специальных символов, нарушающих JSON‑формат.

История не сохраняется:

проверьте права на запись в папке проекта;

убедитесь, что файл history.json не открыт в другом приложении.

Программа не запускается:

проверьте версию Python (должна быть 3.6+);

убедитесь, что все файлы проекта находятся в одной папке;

проверьте, нет ли ошибок в консоли — скопируйте их и пришлите в раздел Issues на GitHub.

Ошибки при сохранении файлов:

убедитесь, что у вас есть права на запись в текущей папке;

попробуйте запустить терминал/командную строку от имени администратора.
